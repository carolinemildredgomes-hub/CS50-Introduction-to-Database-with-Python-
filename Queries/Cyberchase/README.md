# 🚀 Cyberchase SQL Project (CS50)

## 📌 Project Overview

This project is part of **CS50’s Introduction to Databases with SQL**, where I explored and analyzed a real dataset based on the educational TV series *Cyberchase*. The objective was to write efficient SQL queries to extract meaningful insights from a structured database.

The database (`cyberchase.db`) contains a table named `episodes`, which stores detailed information about each episode, including season, title, topic, air date, and production code.

Through this project, I strengthened my ability to **think analytically with data**, **write optimized SQL queries**, and **translate real-world questions into database logic**—skills that are essential in data science, backend development, and AI engineering.

## 🗂️ Database Schema

The `episodes` table includes:

* `id` → Unique identifier for each episode
* `season` → Season number
* `episode_in_season` → Episode position within the season
* `title` → Episode title
* `topic` → Educational concept covered
* `air_date` → Release date (YYYY-MM-DD)
* `production_code` → Internal PBS production ID

---

## ⚙️ Key SQL Concepts Applied

Throughout this project, I applied core SQL concepts, including:

### 🔍 Filtering Data

Used `WHERE` to extract specific rows based on conditions:

* Episodes from a particular season
* Episodes released within a date range
* Episodes with missing topics

### 📊 Aggregation & Counting

Used:

* `COUNT(*)` to measure dataset size
* `COUNT(DISTINCT ...)` to identify unique values

### 📅 Date Handling

Used `BETWEEN` and date comparisons to:

* Analyze episodes across different time periods
* Compare release trends

### 🔎 Pattern Matching

Used `LIKE` to:

* Identify episodes covering specific topics (e.g., fractions)

### 📈 Sorting & Organization

Used `ORDER BY` to:

* Arrange results alphabetically and chronologically

### 🧩 Logical Conditions

Used `AND` / `OR` to:

* Combine multiple conditions for precise querying
